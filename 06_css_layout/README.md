# 06. CSS レイアウト（Flexbox と Position）

ここでは、要素を自由自在に配置するための 2 つの強力な武器を学びます。

これらをマスターすると、「脱初心者」と言えます。

Flexbox と Position は、親要素と子要素を考慮して適切な位置にプロパティを指定する必要があります。そのために `div` 要素を増やす必要があるケースも存在しており、初学者は混乱することが多いです。慌てずに「親と子の関係」を確認しながら進めましょう。

## 1. Flexbox（フレックスボックス）

**「要素を横並びにしたい」と思ったら、迷わずこれを使います。**

かつては `float` という機能を使っていましたが、現在は Flexbox が業界標準です。`float` を使った古いコードを見かけることがあれば、[MDN: float](https://developer.mozilla.org/ja/docs/Web/CSS/float) で調べてみてください。

詳しくは [MDN: Flexbox の基本概念](https://developer.mozilla.org/ja/docs/Web/CSS/CSS_flexible_box_layout/Basic_concepts_of_flexbox) を参照してください。

### 仕組み：親が子を整列させる

Flexbox は、並べたい要素そのものではなく、その **「親要素（コンテナ）」** に命令します。

1. **親**に `display: flex;` をつける → **子要素が横並びになる**
2. **親**に `justify-content` をつける → **横方向**の配置が決まる
3. **親**に `align-items` をつける → **縦方向**の配置が決まる

| プロパティ（親に指定） | 値 | 動作 |
| :--- | :--- | :--- |
| **`justify-content`** | `flex-start`（初期値） | 左寄せ |
| | `center` | 中央寄せ |
| | `space-between` | 両端に配置し、間を均等に空ける（**非常によく使います**） |
| **`align-items`** | `stretch`（初期値） | 高さを親に合わせる |
| | `center` | 上下の真ん中に配置する |
| | `flex-start` | 上揃え |

> **注意**: この「横方向」「縦方向」は、デフォルトの横並び（`flex-direction: row`）の場合です。  
> Flexbox には「主軸（main axis）」と「交差軸（cross axis）」という概念があります。`justify-content` は主軸方向、`align-items` は交差軸方向の配置を制御します。  
> デフォルト（`flex-direction: row`）では主軸が横、交差軸が縦ですが、`flex-direction: column` にすると主軸と交差軸が入れ替わり、`justify-content` が縦方向・`align-items` が横方向を制御するようになります。今は「横並び前提」として覚えておき、縦並びが必要になったときに改めて確認してください。

### gap で要素の間隔を空ける

要素と要素の「間」にだけ余白を作るには `gap` を使います。

```css
.nav-list {
    display: flex;
    gap: 20px; /* 要素と要素の間だけに余白が入る */
}
```

> **`gap` を使う理由**: `margin-right` を各要素に指定すると、最後の要素にも余白が付いてしまいます。  
> `gap` は要素と要素の「間」だけに余白を作るため、最後の要素への余白が自動的に省かれます。

## 2. Position（ポジション）

**「要素を重ねたい」「画面の右下に固定したい」** という時に使います。

通常、HTML の要素は積み木のように重ならずに並びますが、`position` を使うと **浮遊** させることができます。

詳しくは [MDN: position](https://developer.mozilla.org/ja/docs/Web/CSS/position) を参照してください。

### よく使う 3 つの値

| 値 | 説明 |
| :--- | :--- |
| **`relative`** | **相対位置（基準点）**<br>本来の位置から「ちょっとずらす」時に使いますが、実務では **「absolute の基準点」** として親要素につけることがほとんどです。 |
| **`absolute`** | **絶対位置（浮遊）**<br>最も近い `position` が指定された祖先要素の左上を原点 (0, 0) として、好きな場所に配置します。**他の要素と重なります。** |
| **`fixed`** | **固定**<br>画面（ウィンドウ）を基準にして固定します。スクロールしてもついてくる「ヘッダー」や「トップへ戻るボタン」に使います。 |

> **初心者がハマる「基準点」の罠**  
> `absolute` を使いたい時は、**必ずその親（または祖先）要素に `position: relative;` をつけてください。**  
> そうしないと、基準点が「ページの左上」になってしまい、スクロールと共に位置がずれるなど意図しない場所に飛んでいってしまいます。  
> なお `fixed` は「ビューポート（ブラウザの表示領域）の左上」を基準にするため、スクロールしても位置がずれません。この点が `absolute` との大きな違いです。また要素が重なる場合は、`z-index` を調べてみてください（[MDN: z-index](https://developer.mozilla.org/ja/docs/Web/CSS/z-index)）。

## 演習課題：Web サイトのヘッダーとメインビジュアルを作ろう

`index.html` はレイアウトが崩れた状態の Web ページです。
`style.css` を編集して、モダンなレイアウトに直してください。

### 手順 1: ヘッダーを横並びにする（Flexbox）

ヘッダー（`.main-header`）の中に、「ロゴ」と「ナビゲーション（リンク）」が入っていますが、縦に積まれてしまっています。

- `.main-header` に `display: flex;` を指定して横並びにしてください。
- さらに `justify-content: space-between;` を指定して、ロゴを左端、ナビゲーションを右端に離してください。
- `align-items: center;` でロゴとナビゲーションを上下の中央で揃えてください。

ロゴが左端、ナビゲーションが右端に表示されれば正解です。

### 手順 2: ナビゲーション内の横並び（Flexbox）

右側のナビゲーションリスト（`.nav-list`）も縦になっています。

- `.nav-list` に `display: flex;` を指定して横並びにしてください。
- `gap: 20px;` を指定して、リンク同士の間隔を空けてください。

リンクが横並びになり、間隔が空けば正解です。

### 手順 3: 入力フォームの整列（Flexbox）

> **実務メモ：ラベル横並び vs 縦並び**  
> フォームのレイアウトには「ラベルを入力欄の上に置く縦並び」と「ラベルを入力欄の左に置く横並び」の 2 パターンがあります。  
> - **縦並び**：スマートフォンでも崩れにくく、入力欄の幅を広く取れます。シンプルなフォームや項目数が少ない場面に向いています。  
> - **横並び**（このステップ）：各行の意味が一目でわかりやすく、PC 向けの管理画面やビジネスフォームで多く使われます。ただし画面幅が狭いとラベルが見切れるため、後の章で学ぶレスポンシブ対応と組み合わせるのが実務の定石です。

「お問い合わせフォーム」を見てください。

ラベルと入力欄を横並びにしたいですが、エラーメッセージは入力欄の下に出したいです。

- フォームの各行（`.form-row`）に `display: flex;` を指定してください。
- `align-items: flex-start;`（上揃え）を指定してください。
  - ※ `center` にすると、エラーメッセージが出た時にラベルの位置がずれてしまうため、基本は上揃えにします。

> **`flex: 1` について**  
> `style.css` の `.input-group` にはすでに `flex: 1;` が指定されています。これは「フレックスコンテナの残りスペースをすべて使う」という意味です。ラベルの幅（150px）を除いた残りの横幅を、入力欄が自動的に埋めてくれます。

ラベルが左・入力欄が右に並び、エラーメッセージが入力欄の下に表示されれば正解です。

### 手順 4: NEW バッジを画像に重ねる（Position）

カードの右上に「NEW」という赤いバッジ（`.badge`）を乗せたいのですが、今は下に落ちています。

- **親**である `.card-image` に `position: relative;` を指定して、基準点に設定してください。
- **子**である `.badge` に `position: absolute;` を指定して浮かせ、以下の位置指定を追加してください。
  - `top: 0;`（上端に合わせる）
  - `right: 0;`（右端に合わせる）

「NEW」バッジが画像の右上に重なって表示されれば正解です。

### 手順 5: 画面右下にボタンを固定する（Fixed）

「相談する」ボタン（`.floating-btn`）を、スクロールしても常に画面の右下にいるようにします。

- `.floating-btn` に `position: fixed;` を指定してください。
- `bottom: 20px;`、`right: 20px;` で画面の右下隅に配置してください。

スクロールしてもボタンが右下に固定されていれば正解です。

## 覚えなくて良い・後回しにする内容

以下は実務で使う場面がありますが、今の段階では深入りしなくて OK です。

| 内容 | 理由 |
|---|---|
| `flex-direction: column` | 縦並びの Flexbox。横並びを理解した後に自然に覚えられます。 |
| `flex-wrap: wrap` | 画面幅が狭くなったとき折り返す設定。レスポンシブ対応（10章候補）で扱います。 |
| CSS Grid | 縦横 2 次元のレイアウト。Flexbox が使いこなせれば応用で理解できます。詳しくは [MDN: CSS グリッドレイアウト](https://developer.mozilla.org/ja/docs/Web/CSS/CSS_grid_layout) を参照してください。 |
| `position: sticky` | スクロールに応じて `fixed` と `static` を切り替える。`fixed` を理解した後に試してみてください。 |
| `z-index` | `position` が指定された要素の重なり順を制御するプロパティ。要素を重ねるシーンで必要になったときに [MDN: z-index](https://developer.mozilla.org/ja/docs/Web/CSS/z-index) を参照してください。 |

# 10. レスポンシブデザイン（メディアクエリとモバイルファースト）

スマートフォン・タブレット・デスクトップと、ユーザーはさまざまな画面幅でページを閲覧します。画面幅に合わせてレイアウトを変える技術を**レスポンシブデザイン**と呼びます。

この章では、06 章（Flexbox）・07 章（アニメーション）・08 章（CSS 変数）の知識を活用しながら、実務で最頻出の「1列 → 2列 → 3列のカードグリッド」を実装します。

---

## 1. ビューポートと `<meta name="viewport">`

### なぜ必要か

スマートフォンのブラウザは、デフォルトで「PCサイトが 980px 幅で作られている」と想定し、ページ全体を縮小して表示しようとします。その結果、文字が極端に小さくなり、ユーザーはピンチ操作で拡大しなければ読めなくなります。

`<meta name="viewport">` タグを書くと、ブラウザに「デバイスの実際の画面幅でページを表示してほしい」と伝えられます。

### 書き方

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

| 属性値 | 意味 |
|:---|:---|
| `width=device-width` | ビューポートの幅をデバイスの実際の画面幅に合わせる |
| `initial-scale=1.0` | 最初のズーム倍率を 1 倍（等倍）にする |

参考: [MDN: viewport メタタグ](https://developer.mozilla.org/ja/docs/Web/HTML/Viewport_meta_tag)

> **実務メモ**: このリポジトリのすべての `index.html` の `<head>` にはこの1行が最初から書いてあります。実務でも、HTML ファイルを新規作成するときはテンプレートとして最初から書いておくのが一般的です。忘れるとスマートフォンで表示が崩れるため、チェックリストに入れておきましょう。

## 2. メディアクエリ（@media）

### 基本構文

```css
@media (min-width: 768px) {
    /* 768px 以上のときだけ適用されるスタイル */
    .post-card {
        width: calc(50% - 8px);
    }
}
```

`@media` ブロックの外に書いたスタイルは常に適用され、ブロックの中に書いたスタイルは条件を満たしたときだけ上書きされます。

参考: [MDN: @media](https://developer.mozilla.org/ja/docs/Web/CSS/@media)

### モバイルファーストとは何か

CSS を書く順序には2つのアプローチがあります。

| アプローチ | 書き方 | 特徴 |
|:---|:---|:---|
| **モバイルファースト** | 基準スタイル = モバイル。`min-width` で広い画面に上書き | 現代の主流。モバイルから設計する |
| デスクトップファースト | 基準スタイル = デスクトップ。`max-width` で狭い画面に上書き | 古い手法。PCサイトをスマホ対応に後から追加する場合 |

**モバイルファーストをなぜ推奨するか：**

1. **モバイルユーザーが多い**: 現代の Web トラフィックの過半数がスマートフォンからです。
2. **シンプルから複雑へ**: モバイルの1列レイアウトがシンプルな基準で、デスクトップに向けて列数を増やすほうが自然です。
3. **パフォーマンス**: 小さい画面のデバイスは余分なスタイルをダウンロードしなくて済みます。

```css
/* モバイルファースト: まずモバイル用を書く */
.post-card {
    width: 100%; /* モバイルは1列 */
}

/* 768px 以上になったら2列に */
@media (min-width: 768px) {
    .post-card {
        width: calc(50% - 8px);
    }
}

/* 1024px 以上になったら3列に */
@media (min-width: 1024px) {
    .post-card {
        width: calc(33.333% - 11px);
    }
}
```

### ブレークポイントの目安

| ブレークポイント | 対象デバイス |
|:---|:---|
| `768px` | タブレット（iPad ポートレートなど） |
| `1024px` | デスクトップ・ラップトップ |

> **実務メモ**: Figma のデザインデータは通常「モバイル（375px）」「タブレット（768px）」「デスクトップ（1280px）」の3ビューで作成されています。デザイナーとブレークポイントをプロジェクト開始時に合意しておくと、実装時の認識齟齬を防げます。

## 3. Flexbox とメディアクエリの組み合わせ

### `flex-wrap: wrap` + `width` での折り返し制御

Flexbox の `flex-wrap: wrap` を使うと、子要素の幅の合計が親要素を超えたときに自動的に折り返します。この性質を利用して、画面幅ごとに列数を変えられます。

```css
/* 親要素: Flexbox コンテナ */
.post-grid {
    display: flex;
    flex-wrap: wrap;
    gap: 16px;
}

/* 子要素の幅で列数が決まる */
.post-card {
    width: 100%;             /* 1列 */
}

@media (min-width: 768px) {
    .post-card {
        /* gap(16px)を除いた幅を2等分する */
        width: calc(50% - 8px);  /* 2列 */
    }
}

@media (min-width: 1024px) {
    .post-card {
        /* gap(16px × 2)を除いた幅を3等分する */
        width: calc(33.333% - 11px); /* 3列 */
    }
}
```

参考: [MDN: flex-wrap](https://developer.mozilla.org/ja/docs/Web/CSS/flex-wrap)

### `flex-direction: column` でモバイル縦積み

ナビゲーションのようなメニューは、モバイルでは縦に積み重ね、デスクトップでは横に並べます。

```css
/* モバイル: 縦積み */
.nav-list {
    flex-direction: column;
}

/* デスクトップ: 横並び */
@media (min-width: 768px) {
    .nav-list {
        flex-direction: row;
    }
}
```

参考: [MDN: flex-direction](https://developer.mozilla.org/ja/docs/Web/CSS/flex-direction)

> **実務メモ**: 「1列 → 2列 → 3列に切り替わるカードグリッド」は実務で最頻出のパターンです。ブログ・商品一覧・ニュースフィード・検索結果など、あらゆるページで登場します。このパターンを一度しっかり手を動かして覚えると、現場で即使えます。

## 演習課題：ブログ一覧ページをレスポンシブにしよう

`index.html` にブログ記事カードが6枚並んでいます。`style.css` を編集して、画面幅に応じてレイアウトが変わるレスポンシブなページを完成させてください。

### 手順 1: `<meta name="viewport">` の効果を体験する

まず、`<meta name="viewport">` がなかったときの表示を確認します。

1. ブラウザで `index.html` を開きます。
2. DevTools（F12）を開き、デバイスツールバー（`Ctrl+Shift+M` / `Cmd+Shift+M`）をクリックします。
3. デバイス一覧から「iPhone SE」を選択します。
4. ページが iPhone SE の画面幅（375px）で正しく表示されることを確認します。
5. 次に、`index.html` の `<meta name="viewport" ...>` の行をコメントアウトします（`<!-- -->` で囲む）。
6. ページをリロードしてどう変わるか確認します。
7. 確認できたらコメントを戻してください。

viewport あり/なしでページの見た目が変われば正解です。

---

### 手順 2: モバイル（基準）のスタイルを書く

`style.css` の `/* ここに display: flex, ... */` のコメントを参考に、モバイル用のスタイルを設定します。

1. `.post-grid` に以下を追加してください：
   ```css
   display: flex;
   flex-wrap: wrap;
   gap: var(--spacing-md);
   ```

2. `.post-card` の `/* ここに width を設定する */` を以下に変更してください：
   ```css
   width: 100%;
   ```

3. `.site-nav` の `height: 56px;` を削除または `height: auto;` に変更してください。

4. `.nav-list` の `flex-direction` を `column` に変更し、`gap` を `var(--spacing-sm)` にしてください。

iPhone SE の幅（375px）でカードが縦1列・ナビが縦積みになれば正解です。

---

### 手順 3: タブレット（768px 以上）で2列にする

`style.css` の `/* ここに @media (min-width: 768px) { ... } を書く */` の箇所に追記します。

1. 以下の `@media` ブロックを追加してください：
   ```css
   @media (min-width: 768px) {
       .post-card {
           width: calc(50% - var(--spacing-md) / 2);
       }
       .site-nav {
           height: 56px;
       }
       .nav-list {
           flex-direction: row;
           gap: var(--spacing-xl);
       }
   }
   ```

   `calc(50% - var(--spacing-md) / 2)` の意味：
   - 親の幅 100% を2列に分けると1列は 50%
   - ただし `gap: 16px` が1つ入るため、左右のカードがそれぞれ `8px` ずつ引く必要がある
   - → `calc(50% - 8px)` = `calc(50% - var(--spacing-md) / 2)`

2. DevTools のデバイスツールバーで画面幅を 768px に近づけて動作を確認します。

768px 付近でカードが2列・ナビが横並びになれば正解です。

---

### 手順 4: デスクトップ（1024px 以上）で3列にする

`style.css` の `/* ここに @media (min-width: 1024px) { ... } を書く */` の箇所に追記します。

1. 以下の `@media` ブロックを追加してください：
   ```css
   @media (min-width: 1024px) {
       .post-card {
           width: calc(33.333% - var(--spacing-md) * 2 / 3);
       }
   }
   ```

   `calc(33.333% - var(--spacing-md) * 2 / 3)` の意味：
   - 3列に分けると1列は 33.333%
   - `gap: 16px` が2つ入るため、合計 32px を3列で分担 → 1列あたり約 10.67px を引く
   - → `calc(33.333% - 32px / 3)` = `calc(33.333% - var(--spacing-md) * 2 / 3)`

2. DevTools でデスクトップ幅（1024px 以上）を確認します。

1024px 以上でカードが3列になれば正解です。

## 覚えなくて良い・後回しにする内容

| 内容 | 理由 |
|:---|:---|
| CSS Grid によるレイアウト | Flexbox と並ぶもう一つのレイアウト手法。Flexbox を使いこなしてから学ぶ |
| Container Queries（`@container`） | 親要素の幅に応じてスタイルを変える新しい機能。ブラウザサポートは広がっているが、まず `@media` を習得する |
| `clamp()` によるフルードタイポグラフィ | 画面幅に応じてフォントサイズを連続的に変化させる技術。@media を理解してから |
| `prefers-reduced-motion` | アニメーションを抑制するメディアクエリ。動きのある実装をするときに対応する |
| CSS Subgrid | Grid の子要素にグリッドを継承させる機能。CSS Grid を学んでから |
| レスポンシブ画像（`srcset`・`<picture>`） | 画像の解像度・フォーマット切り替え。パフォーマンス最適化として学ぶ |

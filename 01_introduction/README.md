# 01. イントロダクション

ここでは、Web ページが表示される仕組みの概要と、学習に必要なツールの使い方を学びます。

## HTML と CSS の概要

Web ページは主に 2 つの言語で作られています。

### HTML (HyperText Markup Language)

**「Web ページの骨組み」** を作る言語です。<br>
「ここは見出し」「ここは段落」「ここは画像」といった文書の構造を定義します。<br>
タグ（`< >`）を使って囲むことで意味を持たせます。

> **用語解説：HyperText（ハイパーテキスト）とは？**<br>
> 「Hyper（超越した）Text（文書）」という意味です。<br>
> 普通の本は 1 ページ目から順に読みますが、Web ページはリンクをクリックして別のページへ瞬時に移動できます。この「文書と文書がつながる仕組み」のことを指します。

### CSS (Cascading Style Sheets)

**「Web ページの見た目」** を作る言語です。<br>
HTML で作った骨組みに対して、「文字を赤くする」「背景をグレーにする」「中央に配置する」といった装飾（スタイル）を指定します。

> **用語解説：Cascading（カスケーディング）とは？**<br>
> 「階段状の滝」という意味です。<br>
> CSS には「親の設定が子に引き継がれる」「後から書いた設定が優先される」といったルールがあります。この「上流から下流へスタイルが流れていく仕組み」を滝に例えています。

## 学習ツールの使い方

### 1. Web サーバーの確認

すでにルートディレクトリ（この場合はプロジェクト直下のこと。もう一つ上のフォルダのこと。）で `./run_web_server.sh` を実行しているはずです。
![サーバー起動後の画面](/assets/img/01_server_preview.png)

もしブラウザを閉じてしまった場合は、ターミナルの通知にある「ポート」タブにある「転送されたアドレス」のURLをクリックしてください。
![サーバー起動後の画面](/assets/img/01_server_reopen.png)



### 2. 開発者ツール（DevTools）の操作

ブラウザには、表示されているページのコードを確認・編集できる「開発者ツール」がついています。これを使って学習を進めます。

1. **ツールの起動**<br>
   ブラウザ画面上で **右クリック** し、メニューから **「検証」** を選択します（または `F12` キー）。
   ![開発者ツールのElementsタブとStylesタブ](/assets/img/02_show_devtools.png)

2. **HTML を見る（Elements タブ）**<br>
   「Elements（要素）」タブを見てください。`index.html` に書かれた内容がそのまま表示されています。
   `<body>` タグの `▶` をクリックして開くと、中身が見えます。
   ![開発者ツールのElementsタブとStylesタブ](/assets/img/03_devtools_elements_tab.png)
3. **CSS を見る（Styles タブ）**<br>
   Elements タブで `<h1>` タグをクリックして選択してください。
   「Styles（スタイル）」タブに、`color: blue;` などの指定が表示されます。
   ![開発者ツールのElementsタブとStylesタブ](/assets/img/04_devtools_elements_style.png)

> **補足：日本語化について**<br>
> Chrome の設定により英語で表示されることがありますが、そのままでも問題ありません。<br>
> エラーメッセージなどは英語で検索したほうが情報が見つかりやすいため、**「英語のまま慣れる」** のもエンジニアへの近道です。<br>
> （どうしても日本語にしたい場合は、DevTools 右上の歯車アイコン ⚙️ > `Preferences` > `Language` から変更できます）

### 3. 一時的な変更を試す

開発者ツール上で値を書き換えると、リアルタイムで画面に反映されます。

1. Styles タブの `color: blue;` の色の部分をクリックします。
   ![開発者ツールで色を変更する前の様子](/assets/img/03_style_change_before.png)
2. 値に `red` と入力して Enter を押してください。
3. 画面の「ようこそ」という文字が赤くなったことを確認します。
   ![開発者ツールで色を変更した後の様子](/assets/img/04_style_change_after.png)
4. **ブラウザの再読み込みボタン** を押してください。<br>
   -> 元の青色に戻ります。

> **ポイント:**<br>
> 開発者ツールでの変更は **一時的** です。<br>
> 本当に変更したい場合は、VS Code で `style.css` ファイルを編集して保存する必要があります。

## 注意：見る環境によって「見え方」は変わります

HTML と CSS は、どのパソコンやスマホで見ても**完全に同じ表示になるわけではありません。**

かつて（Internet Explorer 6, 7, 8 などが主流だった時代）は、ブラウザごとの表示崩れが非常に激しく、Web エンジニアはブラウザごとの修正対応に多くの時間を費やしていました（「CSS ドラえもん」で インターネット検索してもらうと画像が出てきます）。

現在はブラウザの規格が統一されてきたため、昔ほど大きな差はなくなりましたが、それでも以下の要因によって表示に差異（ズレ）が発生します。

1. **レンダリングエンジンの違い**.<br>
   Web ブラウザがコードを読み取って画面に絵を描くための「エンジン（Chrome 系, Safari 系, Firefox 系など）」が異なるため、解釈に微妙な差が出ます。<br>
   「エンジン」と言われても想像しにくいかもしれませんが、「料理人」だと仮定してみてください。同じ材料を渡して「カレー」を作ってほしいと依頼しても、材料のサイズや調理行程、味付けの仕方まですべてが同じとは限らないですよね？
2. **ディスプレイの DPI（解像度）の違い**<br>
   OS やモニターの画素密度によって、文字の太さや線の鮮明さが変わることがあります。<br>
   同じフォントサイズが `10px` と明記されていても、そもそも `1px` = `何ドット` なのかは、`DPI`によって異なります。それによって横幅のサイズ解釈が異なるケースがあり、スマホだけ異なる表示になることがあります。

### 仕事での心構え

実務で Web サイトを開発する際は、 **「サポートするデバイス（PC, iPhone, Android など）」と「ブラウザ（Chrome, Safari, Edge など）」** のすべてで正しく表示されるかを確認する必要があります。

> **この学習リポジトリでの扱い**<br>
> 今回の学習では、基礎をスムーズに学ぶため **「PC 版の Google Chrome」** での表示を基準に進めます。スマホ表示（レスポンシブ対応）などの複雑な調整は範囲外としていますが、実務では「様々な環境でのチェックが必要である」ということを頭の片隅に置いて学習を進めてください。

## コラム：エンジニアの小話

### 1. HTML はプログラミングではない？

よく「HTML はプログラミング言語ではない」と言われることがあります。これは、HTML が「計算」や「条件分岐（もし〜なら）」といった処理を行わないためです。

HTML は **「マークアップ（印をつける）言語」** です。
Word や Excel で「ここは太字」「ここはタイトル」と設定する感覚に近く、コンピュータに対して「文書の構造」を伝えるための **設定ファイル（仕様書）** のようなものだと捉えてください。

ただし、プログラミングではないからと言って「簡単」「重要ではない」という意味ではありません。すべての Web システムの土台となる重要な技術です。

### 2. 再び学習することになる

初学者のうちは、HTML も直に簡単に見えてくるかもしれません。しかし、学習が進んで CSS（デザイン）や JavaScript（機能）が複雑になってくると、必ず壁にぶつかります。

- 「CSS が思うように効かない…」
- 「JavaScript で要素がうまく取得できない…」
- 「別のプログラムから抜粋してきたのに思った結果と違う…」

その原因の中には、実は **「HTML の構造が悪いこと」** にあります。

効率よく開発するためには、実は HTML の設計（タグの選び方や ID/Class の付け方）が最も重要だと気づく日が来ます。その時、あなたはもう一度、より深いレベルで HTML を学び直すことになるでしょう。

# 05. CSS の基礎（書き場所・色・ボックスモデル）

ここでは、CSS を使って Web ページの見た目を整えるための基本ルールを学習します。

CSS はプロパティ（設定項目）が膨大にありますが、すべて暗記する必要はありません。

「調べ方」と「書き場所のルール」を覚えましょう。

詳しくは [MDN: CSS リファレンス](https://developer.mozilla.org/ja/docs/Web/CSS/Reference) を参照してください。

## 1. CSS の書き場所とセレクタ

CSS はスタイルが「競合したとき、特定のルールで一つが選ばれる」仕組みを持ちます。この「上流から下流へ流れ、途中で上書きされる」様子から **Cascading**（カスケーディング）と呼ばれています。「CSS が思い通りに効かない」と感じたら、「どのスタイルが上書きしているか」を疑ってください。

まず、「CSS をどこに書くか」ですが、大きく分けて次の２つの方法があります。

- 「HTML ファイル内に書く（`<style>`タグ）」
- 「別の CSS ファイルに書いて読み込む（`.css`ファイル）」

実務では、管理しやすくするために 「別の CSS ファイル（`style.css`など）」 に書いて、HTML から読み込むのが基本です。

CSS ファイルを読み込むには、HTML の `<head>` タグ内に次のように書きます。

```html
<link rel="stylesheet" href="style.css" />
```

- `rel="stylesheet"` は「このファイルはスタイルシートです」という意味です。
- `href="style.css"` は読み込む CSS ファイルのパスを指定します。

スタイルを適用する方法は主に 3 つありますが、実際の開発現場では使い分けが明確に決まっています。

### ① タグに指定する（Tag Selector）

HTML のタグそのもの (`h1`, `p`, `body` など) に指定します。

- **書き方:** CSS ファイルに `h1 { color: red; }` と書きます。
- **現場での扱い:** **「初期設定」以外ではあまり使いません。**
  - 理由: すべての `h1` が変わってしまうため、影響範囲が大きすぎるからです。プロジェクトの最初に全体デザインを決める時だけ使います。多くの場合、Web ブラウザ間の差を埋める目的でリセット CSS として宣言されています。

### ② クラスに指定する（Class Selector）

HTML に `class="btn"` と名前をつけ、CSS ファイルの `.btn` に対してスタイルを指定します。

- **書き方:** `.btn { color: red; }` （先頭にドット `.` をつける）
- **現場での扱い:** **これが基本です。9 割はこれで書きます。**
  - 理由: 名前をつけた場所にだけ適用できるため、管理しやすいからです。通常は共通の CSS ファイルを用意して一元管理します。

### ③ タグに直接書く（Inline Style）

HTML タグの `style` 属性に直接書きます。

- **書き方:** `<p style="color: red;">`
- **現場での扱い:** **原則として使用しません。**
  - 理由: CSS ファイルで一元管理できなくなるためです。開発中の「お試し」で使うことがある程度です。

### セレクターの優先順位（詳細度）

複数のスタイルが同じ要素に当たるとき、**「より具体的な指定が勝つ」** ルールがあります。

| セレクター | 優先度（低→高） |
|---|---|
| タグ (`h1 {}`) | 低 |
| クラス (`.btn {}`) | 中 |
| ID (`#header {}`) | 高 |
| インライン (`style=""`) | 最高 |

> **実務メモ**: 実務では **クラスセレクターだけ** で統一するのが慣習です。
> ID セレクターは詳細度が高すぎて後から CSS で上書きしにくくなるため、CSS での使用は避けてください。
> 「なぜか CSS が効かない」の多くはこの優先順位が原因です。

詳しくは [MDN: 詳細度](https://developer.mozilla.org/ja/docs/Web/CSS/Specificity) を参照してください。

## 2. プロパティと調べ方（MDN）

CSS は `プロパティ: 値;` の形式で書きます。

プロパティは星の数ほどあるため、すべてを暗記することは不可能です。

**「やりたいこと + MDN」** で検索する癖をつけましょう。

例：「文字を太くしたい」→ `font bold MDN` で検索 → `font-weight` を見つける

- **[MDN CSS リファレンス（プロパティ一覧）](https://developer.mozilla.org/ja/docs/Web/CSS/Reference)**

### よく使う単位

サイズを指定する単位もたくさんありますが、まずは以下を覚えましょう。

- **`px` (ピクセル):** 画面上のドット数。「絶対にこのサイズにする」という時に使います。
- **`%` (パーセント):** 親の要素に対する割合。「画面の半分 (50%)」などの時に使います。
- **`rem`（ルート em）:** HTML のルート要素の文字サイズ（通常 16px）を基準にした倍率。`1rem = 16px`・`1.5rem = 24px` となります。フォントサイズの指定によく使われます。  
  ブラウザのアクセシビリティ設定（ユーザーが文字サイズを大きくする設定）に連動するため、`px` で固定すると設定が無視されてしまいます。フォントサイズには `rem` を使うのが現場の慣習です。

> **今は深入りしなくて OK**: まず `px` で動くものを作れるようになることが先決です。
> `rem` の使い分けは CSS レイアウトを学んだ後に理解しやすくなります。

### 色の指定方法（カラーコード）

色は `red` や `blue` といった名前でも指定できますが、実務では **16 進数カラーコード** (`#` + 6 桁の英数字) を使います。

「HTML カラーコード」で検索すると色見本サイトが見つかります。

また、実際の現場ではプロダクトごとにテーマカラーがあるため、指定された色コードを使います。

- **`#000000`**: 黒
- **`#FFFFFF`**: 白
- **`#FF0000`**: 赤 (Red が MAX)
- **`#333333`**: 濃いグレー（文字色によく使われます）

## 3. ボックスモデル（Web の箱の構造）

Web ページの要素はすべて「四角い箱」でできています。

この箱の構造を **ボックスモデル** と呼びます。

詳しくは [MDN: ボックスモデル](https://developer.mozilla.org/ja/docs/Learn_web_development/Core/Styling_basics/Box_model) を参照してください。

| 名称 | CSS プロパティ | イメージ | 役割 |
| :--- | :--- | :--- | :--- |
| **コンテンツ** | `width`, `height` | **中身**（プレゼント） | 文字や画像そのものの領域。 |
| **パディング** | `padding` | **緩衝材**（プチプチ） | 中身と枠線の間の隙間。**背景色はここにも塗られます。** |
| **ボーダー** | `border` | **段ボール**（箱） | 領域の境界線。 |
| **マージン** | `margin` | **ソーシャルディスタンス** | 隣の箱との距離。**背景色は塗られません（透明）。** |

> **重要：**  
> 「文字と枠線の間を広げたい」なら **padding**  
> 「隣のボタンと離したい」なら **margin**

DevTools で要素を選択すると、`margin`（橙）・`border`（黄）・`padding`（緑）・`content`（青）がボックスモデルとして色分けして表示されます。「なぜこの余白が生まれているのか」を調べるときに活用してください。

### box-sizing: border-box（重要）

デフォルトでは、`width: 300px` と指定しても、`padding` と `border` の分だけ実際の幅が広がってしまいます。

```css
/* デフォルト (content-box): width に padding と border が加算される */
.box { width: 300px; padding: 10px; border: 1px solid; }
/* 実際の幅: 300 + 10×2 + 1×2 = 322px */

/* border-box: width の中に padding と border を含める */
* { box-sizing: border-box; }
.box { width: 300px; padding: 10px; border: 1px solid; }
/* 実際の幅: 300px のまま */
```

実務では `* { box-sizing: border-box; }` を必ずファイルの先頭に書きます。これがないと幅の計算が複雑になります。

## 演習課題：商品カードをデザインしよう

`index.html` を開くと、スタイルが当たっていない「商品カード」が表示されます。

`style.css` を編集して、以下のデザイン指示通りに仕上げてください。

### 手順 1: 文字の装飾

商品名（`.product-title`）を目立たせます。MDN でプロパティを調べながらやってみましょう。

- 文字の大きさを `24px` にしてください (`font-size`)。
- 文字の色を `#333333`（濃いグレー）にしてください (`color`)。
- 太字にしてください (`font-weight: bold`)。

商品名が大きく・濃く・太字で表示されれば正解です。

> **DevTools で CSS を確認する（実務の基本操作）**  
> ブラウザで要素を右クリック →「検証」を開き、Elements パネルで `.product-title` を選択します。  
> 右側の **Styles パネル** に適用されている CSS プロパティが一覧表示されます。  
> 打ち消し線が引かれているプロパティは「別のルールに上書きされている」サインです。  
> CSS が効かないと感じたらまず Styles パネルで「どのルールが当たっているか」を確認する習慣をつけましょう。

### 手順 2: パディングで内側を広げる

「購入する」ボタン（`.btn-buy`）を見てください。文字と枠線がキツキツです。

- 上下に `10px`、左右に `20px` の **padding** を追加して、ボタンをふっくらさせてください。
- 背景色を `#e67e22`（オレンジ）にしてください (`background-color`)。
- 文字色を `#ffffff`（白）にしてください。

ボタンにオレンジの背景と余白が付き、押しやすくなれば正解です。

### 手順 3: ボーダーで枠線をつける

カード全体（`.product-card`）に枠線をつけます。

- 太さ `1px`、実線（`solid`）、色 `#dddddd`（薄いグレー）の **border** を指定してください。
- カードの幅（`width`）を `300px` に指定してください。

カードに薄いグレーの枠線が付き、幅が `300px` になれば正解です。

> **補足：`margin: 0 auto` でのセンタリング**  
> `style.css` にはすでに `.product-card { margin: 0 auto; }` が書かれています。  
> これは「上下のマージンを 0、左右のマージンを自動（均等）にする」という指定で、要素を水平方向に中央揃えします。  
> ただし、**`width` が指定されていないと機能しません。** `width` がないとブロック要素は親要素いっぱいに広がるため、左右に余白を作る余地がないからです。  
> 「`width` を設定してから `margin: 0 auto`」がセンタリングの基本パターンです。

### 手順 4: マージンで距離をとる

カードの中の要素が全部くっついていて窮屈です。

- 価格（`.product-price`）の下に `20px` の **margin** を入れて、ボタンとの距離を離してください。

価格とボタンの間に空白が生まれれば正解です。

## コラム：文字がはみ出してしまう時は？

初心者がよくやる失敗に、**「高さ (`height`) を固定してしまう」** というものがあります。

```css
/* 悪い例 */
.box {
  height: 50px; /* 高さを固定 */
}
```

もしこの中に、50px を超える長い文章が入ってきたらどうなるでしょうか？

文章は箱からあふれ出し、下の要素と重なって表示が崩れてしまいます。これを **Overflow（オーバーフロー）** と言います。

> **解決策:**  
> 基本的には `height`（高さ）は指定しないでください。指定しなければ、中身が増えても箱が自動的に縦に伸びてくれます。

## 覚えなくて良い・後回しにする内容

以下は実務で使う場面がありますが、今の段階では深入りしなくて OK です。

| 内容 | 理由 |
|---|---|
| `em` ユニット | 親要素のフォントサイズを基準にする単位。`rem` を使えば十分なため、後回しで OK です。 |
| CSS カスタムプロパティ（変数） | `--main-color: #333;` のように色や値を変数化できる機能。デザイン基礎（08章候補）で扱います。 |
| `rgba()` / `hsl()` 色指定 | 透明度付きの色指定や色相ベースの指定。まず `#` カラーコードを使えることが先決です。 |
| `!important` | CSS の優先順位を強制上書きする書き方。**使ってはいけません。** `!important` で上書きしようとすると、さらに強い `!important` が必要になり（`!important` スパイラル）、最終的に CSS 全体が管理不能になります。 |

# 07. CSS の高度なテクニック（アニメーションと状態変化）

HTML/CSS 学習の総仕上げです。Web ページに「動き」をつけて、ユーザーにとって使い心地の良い UI を作りましょう。

詳しくは [MDN: CSS アニメーションの使い方](https://developer.mozilla.org/ja/docs/Web/CSS/CSS_animations/Using_CSS_animations) を参照してください。

## 1. 状態の変化（疑似クラス）

要素は「常に同じ姿」ではありません。ユーザーの操作によって状態が変わります。

CSS では `:` を使って、「〜している時」のデザインを指定できます。詳しくは [MDN: 疑似クラス](https://developer.mozilla.org/ja/docs/Web/CSS/Pseudo-classes) を参照してください。

| 疑似クラス    | 説明                           | よく使う場面                                                   |
| :------------ | :----------------------------- | :------------------------------------------------------------- |
| **`:hover`**  | **マウスが乗っている時**       | ボタンの色を変える、画像を少し拡大する。                       |
| **`:active`** | **クリックして押されている時** | ボタンを沈み込ませる（押した感）。                             |
| **`:focus`**  | **キーボードで選択された時**   | 入力欄やボタンに枠線をつける。キーボード操作のアクセシビリティに必須。 |

> **実務メモ**: キーボードで Tab キーを押して操作するユーザーにとって、`:focus` のスタイルは「今どこにいるか」を示す唯一の手がかりです。「デザインが邪魔」という理由で `:focus { outline: none; }` と消してしまうと、アクセシビリティ上の問題になります。現場のコードレビューでよく指摘される点です。

## 2. 時間の変化（トランジション）

`:hover` で色を変える時、パッと切り替わると機械的で安っぽく見えます。

`transition` プロパティを使うと、変化に「時間」をかけて滑らかにできます。詳しくは [MDN: transition](https://developer.mozilla.org/ja/docs/Web/CSS/transition) を参照してください。

```css
.btn {
    background-color: blue;
    /* 変化にかける時間を 0.3秒 に設定 */
    transition: 0.3s;
}

.btn:hover {
    background-color: red; /* 0.3秒かけて青から赤になる */
}
```

## 3. アニメーション（キーフレーム）

ユーザーの操作に関係なく、ずっと動き続けるもの（ローディングなど）を作るには `@keyframes` を使います。詳しくは [MDN: @keyframes](https://developer.mozilla.org/ja/docs/Web/CSS/@keyframes) を参照してください。

パラパラ漫画のように「開始（0%）」と「終了（100%）」の状態を定義します。

```css
/* アニメーションの動き方を定義 */
@keyframes フェードイン {
    0%   { opacity: 0; }  /* 開始: 透明 */
    100% { opacity: 1; }  /* 終了: 不透明 */
}

/* 要素に適用 */
.box {
    animation: フェードイン 1s ease;
    /*         名前         時間  速度カーブ */
}
```

アニメーションでよく使う `transform` プロパティを使うと、要素を移動・回転・拡縮できます。詳しくは [MDN: transform](https://developer.mozilla.org/ja/docs/Web/CSS/transform) を参照してください。

| 関数 | 説明 |
| :--- | :--- |
| `translateY(-10px)` | 上に 10px 移動（マイナスが上） |
| `rotate(360deg)` | 360 度回転 |
| `scale(1.05)` | 1.05 倍に拡大 |

## 演習課題：リッチなボタンとローディングを作ろう

`index.html` にある無機質な要素たちに、CSS で命を吹き込んでください。

### 手順 1: ホバーで浮き上がるカード (:hover, box-shadow)

商品カード（`.card`）に動きをつけます。

1. `.card` に `transition: 0.3s;` を追加して、変化を滑らかにする準備をします。

2. `.card:hover`（ホバー時）に以下を設定してください。

   - `transform: translateY(-10px);`（上に 10px 移動する）
   - `box-shadow: 0 10px 20px rgba(0,0,0,0.2);`（影を濃く広げる）

カードにマウスを乗せた時、上に移動して影が広がれば正解です。

### 手順 2: 押せる立体ボタン (:active, box-shadow)

送信ボタン（`.btn-push`）を、クリックした時だけ沈み込ませます。

CSS ですでに「下の影（`border-bottom`）」で立体感を出しています。

1. `.btn-push:active`（押した時）に以下を設定してください。

   - `border-bottom: none;`（下の影を消す）
   - `transform: translateY(4px);`（影の分だけ下にずらす）

ボタンをクリックした瞬間、下に沈み込む感触が出れば正解です。

### 手順 3: くるくる回るローディング (@keyframes)

ローディングアイコン（`.spinner`）を回転させます。

1. `style.css` の一番下にある `@keyframes spin` の中身を完成させてください。

   - `0% { transform: rotate(0deg); }`（開始: 回転なし）
   - `100% { transform: rotate(360deg); }`（終了: 1 回転）

2. `.spinner` クラスに `animation: spin 1s linear infinite;` を追加して、アニメーションを適用してください。

   - `spin`：上で定義したアニメーション名
   - `1s`：1 秒で 1 周
   - `linear`：等速で回転（`ease` だと加速・減速する）
   - `infinite`：無限に繰り返す

水色の円が途切れなく回り続ければ正解です。

## 覚えなくて良い・後回しにする内容

| 内容 | 理由 |
|---|---|
| `transition-timing-function`（`ease-in-out` など） | 変化の緩急を細かく制御する設定。今は `0.3s` で十分。 |
| `animation-delay` / `animation-fill-mode` | 複数アニメーションの細かい制御。基本を掴んだ後に。 |
| `:focus-visible` | `:focus` のうちキーボード操作時だけ表示する高度な制御。09 章のアクセシビリティで詳しく扱います。 |
| CSS カスタムプロパティ（変数） | `--color: red;` のように値を変数として管理する機能。次の 08 章で学習します。 |

## Appendix：さらにステップアップするために

次の章（08〜10章）では、今回学んだ CSS の知識をさらに広げていきます。

### 1. クラス名の命名規則（BEM など）

大規模な開発では `class="red-text"` のような単純な名前は使いません（他の場所で意図せず色が変わり、事故になるため）。

`Block__Element--Modifier` のように、構造化された名前をつける **BEM** というルールなどが採用されることが多いです。

### 2. CSS フレームワーク

画面などの開発では、CSS をゼロから書くことは少なく、**Bootstrap** や **Tailwind CSS** といった「既存のパーツ集」を使うことが一般的です。

ただし、微調整には今回学んだ `margin`, `padding`, `flex` の知識が必須となります。

### 3. リセット CSS

Web ブラウザ（`Chrome`, `Safari`, `Edge` など）には、最初から「デフォルトの CSS（ユーザーエージェントスタイル）」が組み込まれています。

何も CSS を書いていないのに、`<h1>` の文字が大きかったり、`<ul>` に点（・）がついたり、`<body>` に少し隙間（マージン）があったりするのはこのためです。

このデフォルトのスタイルは **ブラウザごとに微妙に異なります**。「Chrome では綺麗なのに、Safari だとズレる」といった問題を避けるため、プロの現場では CSS ファイルの先頭で「リセット CSS」を読み込むのが一般的です。実務では `Destyle.css` や `Normalize.css` といったライブラリを使います。

### 4. おすすめの学習方法：模写（もしゃ）コーディング

基礎が身についたか確認する最高の方法は、**「既存のサイトを見て、同じ見た目を作ってみる」** ことです。

好きな企業のランディングページなど、実物を見ながらコードを書いてみると、圧倒的に力がつきます。

### 5. MDN 精読のすすめ

[MDN](https://developer.mozilla.org/ja/) には HTML・CSS・JavaScript について仕様と注意点が網羅的に記載されています。

今後の学習でも業務でも、不明点を検索して調べることになります。自分で簡単な Web ページを自由に作成できるまで、MDN を積極的に読むことをおすすめします。

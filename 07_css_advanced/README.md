# 07. CSS の高度なテクニック（アニメーションと状態変化）

HTML/CSS 学習の総仕上げです。
Web ページに「動き」をつけて、ユーザーにとって使い心地の良い（リッチな）UI を作りましょう。

## 1. 状態の変化（疑似クラス）

要素は「常に同じ姿」ではありません。ユーザーの操作によって状態が変わります。
CSS では `:` を使って、「〜している時」のデザインを指定できます。

| 疑似クラス    | 説明                           | よく使う場面                             |
| :------------ | :----------------------------- | :--------------------------------------- |
| **`:hover`**  | **マウスが乗っている時**       | ボタンの色を変える、画像を少し拡大する。 |
| **`:active`** | **クリックして押されている時** | ボタンを沈み込ませる（押した感）。       |

## 2. 時間の変化（トランジション）

`:hover` で色を変える時、パッと切り替わると機械的で安っぽく見えます。
`transition` プロパティを使うと、変化に「時間」をかけて滑らかにできます。

```css
.btn {
  background-color: blue;
  /* 変化にかける時間を 0.3秒 に設定 */
  transition: 0.3s;
}

.btn:hover {
  background-color: red; /* 0.3秒かけて青から赤になる */
}
```

## 3. アニメーション（キーフレーム）

ユーザーの操作に関係なく、ずっと動き続けるもの（ローディングなど）を作るには `@keyframes` を使います。 パラパラ漫画のように「開始」と「終了」の状態を定義します。

## 演習課題: リッチなボタンとローディングを作ろう

index.html にある無機質な要素たちに、CSS で命を吹き込んでください。

### 手順 1: ホバーで浮き上がるカード (:hover, box-shadow)

商品カード（`.card`）に動きをつけます。

1. `.card` に `transition: 0.3s;` を追加して、変化を滑らかにする準備をします。
1. `.card:hover`（ホバー時）に以下を設定してください。<br>
   `transform: translateY(-10px);` （上に 10px 移動する）<br>
   `box-shadow: 0 10px 20px rgba(0,0,0,0.2);` （影を濃く広げる）

-> 「ふわっ」と浮いたように見えますか？

### 手順 2: 押せる立体ボタン (:active, box-shadow)

送信ボタン（`.btn-push`）を、クリックした時だけ沈み込ませます。 `CSS` ですでに「下の影（`border-bottom`）」で立体感を出しています。

1. `.btn-push:active`（押した時）に以下を設定してください。<br>
   `border-bottom: none;` （下の影を消す）<br>
   `transform: translateY(4px);` （影の分だけ下にずらす）

-> カチッと押したような物理的な感覚が出ますか？

### 手順 3: くるくる回るローディング (@keyframes)

ローディングアイコン（`.spinner`）を回転させます。

1. `style.css` の一番下にある `@keyframes spin` の中身を完成させてください。
   - 0%（開始）は `transform: rotate(0deg);`
   - 100%（終了）は `transform: rotate(360deg);`
1. `.spinner` クラスに `animation: spin 1s linear infinite;` を追加して、アニメーションを適用してください。

## Appendix：さらにステップアップするために

このリポジトリを修了した方向けに、現場で役立つ周辺知識やキーワードを紹介します。

### 1. スマホ対応（レスポンシブデザイン）

現在は PC よりもスマホで閲覧されることが多い時代です。

画面の幅に合わせてレイアウトを切り替える技術を **「メディアクエリ (`@media`)」** と呼びます。

「画面幅が狭くなったら、横並び（Flexbox）を解除して縦並びにする」といった手法が一般的です。

### 2. クラス名の命名規則（BEM など）

大規模な開発では `class="red-text"` のような単純な名前は使いません（他の場所で意図せず色が変わり、事故になるため）。

`Block__Element--Modifier` のように、構造化された名前をつける **BEM** というルールなどが採用されることが多いです。

### 3. CSS フレームワーク

画面などの開発では、CSS をゼロから書くことは少なく、**Bootstrap** や **Tailwind CSS** といった「既存のパーツ集」を使うことが一般的です。

ただし、微調整には今回学んだ `margin`, `padding`, `flex` の知識が必須となります。

### 4. リセット CSS

Web ブラウザ（`Chrome`, `Safari`, `Edge` など）には、最初から 「デフォルトの CSS（ユーザーエージェントスタイル）」 が組み込まれています。

何も CSS を書いていないのに、`<h1>` の文字が大きかったり、`<ul>` に・（中黒）がついたり、`<body>` に少し隙間（マージン）があったりするのはこのためです。

しかし、このデフォルトのスタイルは **ブラウザごとに微妙に異なります**。

「Chrome では綺麗なのに、Safari だとズレる」といった問題を避けるため、プロの現場では CSS ファイルの先頭（または専用のファイル）で 「リセット CSS」 を読み込むのが常識となっています。

**リセット CSS の役割**
ブラウザが持っているデフォルトのスタイルを全て打ち消し（リセットし）、どのブラウザで見ても「同じ更地の状態」からスタートできるようにすること です。

今回の学習では基礎を理解するためにあえて使いませんでしたが、実務では `Destyle.css` や `Normalize.css` といった有名なリセット CSS ライブラリを使うことが一般的です。

また、一般的な CSS フレームワークにはこれが組み込まれています。

### 5. おすすめの学習方法：模写（もしゃ）コーディング

基礎が身についたか確認する最高の方法は、**「既存のサイトを見て、同じ見た目を作ってみる」** ことです。

Google のトップページや、好きな企業のランディングページなど、実物を見ながらコードを書いてみると、圧倒的に力がつきます。

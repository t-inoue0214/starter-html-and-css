# 04. HTML フォームの部品（UI カタログ）

ここでは、ユーザーからデータを受け取るための様々な「入力部品（パーツ）」の使い方を学習します。

詳しくは [MDN: Web フォームの構築](https://developer.mozilla.org/ja/docs/Learn_web_development/Extensions/Forms) を参照してください。

## 1. label（ラベル）タグの重要性

フォームを作る時、ただ入力欄 (`<input>`) を置くだけでは不十分です。

「この入力欄は何のためのものか」を示す文字と、入力欄を **紐付ける** 必要があります。

```html
<label for="user-name">お名前：</label>
<input type="text" id="user-name" />
```

### 紐付けるメリット

**1. ヒットエリアが広がる**

チェックボックスなどの小さな四角形を正確にクリックするのは意外と大変です。  
ラベルと紐付けることで「クリックできる範囲」が文字の分まで広がり、ユーザーがストレスなく操作できるようになります。

**2. スクリーンリーダーに伝わる**

スクリーンリーダー（視覚障害者が使う読み上げソフト）は `for`/`id` の紐付けを頼りに「この入力欄は〇〇のためのものです」と読み上げます。  
紐付けがないと「テキスト入力欄」とだけ読み上げられ、何を入力すればよいかが伝わりません。

### label の書き方：2 パターン

どちらも動作は同じです。チームのコーディング規約に合わせて統一してください。

```html
<!-- パターン1: 紐付け型（for と id を対応させる） -->
<input type="checkbox" id="agree">
<label for="agree">同意する</label>

<!-- パターン2: 内包型（input を label で囲む） -->
<label>
  <input type="checkbox"> 同意する
</label>
```

## 2. よく使うパーツ一覧

詳しくは [MDN: input 要素](https://developer.mozilla.org/ja/docs/Web/HTML/Element/input) を参照してください。

テキスト入力系

| タグ・属性 | 説明 |
| -- | -- |
| `<input type="text">` | 1 行テキスト。普通の文字入力に使います。 |
| `<input type="password">` | パスワード。入力した文字が ●●● で隠されます。 |
| [`<textarea>`](https://developer.mozilla.org/ja/docs/Web/HTML/Element/textarea) | 複数行テキスト。長文の感想やお問い合わせ内容に使います。`</textarea>` の閉じタグが必要です。 |

選択系

| タグ・属性 | 説明 |
| -- | -- |
| `<input type="radio">` | ラジオボタン。「A か B か」どれか 1 つを選ばせるときに使います。同じ `name` 属性をつけることでグループ化できます。 |
| `<input type="checkbox">` | チェックボックス。複数選択の場合に使います。 |
| [`<select>`](https://developer.mozilla.org/ja/docs/Web/HTML/Element/select) & `<option>` | セレクトボックス。プルダウン形式で選択肢を表示します。 |

## 3. バリデーション属性（入力チェック）

ブラウザが自動でチェックできる属性があります。送信ボタンを押したとき、条件を満たしていない場合はブラウザがエラーメッセージを表示します。

| 属性 | 説明 |
|---|---|
| `required` | 必須入力。空のまま送信しようとするとエラーが出ます。 |
| `minlength` / `maxlength` | 文字数の最小 / 最大を制限します。 |
| `min` / `max` | `type="number"` の入力範囲を制限します。 |

詳しくは [MDN: クライアントサイドフォーム検証](https://developer.mozilla.org/ja/docs/Learn_web_development/Extensions/Forms/Form_validation) を参照してください。

## 4. button の type 属性

[`<button>`](https://developer.mozilla.org/ja/docs/Web/HTML/Element/button) タグには必ず `type` を指定してください。

| type | 動作 |
|---|---|
| `type="submit"` | フォームの内容をサーバーへ送信します（省略時のデフォルト）。 |
| `type="button"` | 何もしません。JavaScript で動作を定義するときに使います。 |

> **実務メモ**: `<form>` の中に `<button>` を置くとき、`type` を省略すると自動的に `type="submit"` になります。  
> クリックするたびにページがリロードされる「謎のバグ」の原因になりやすいため、`type` は必ず明示してください。
>
> **よくあるバグ例**: 入力フォームの中に「×（モーダルを閉じる）」ボタンを追加したとき、`type` を書き忘れると、×ボタンを押した瞬間にフォームが送信されてページがリロードされます。「閉じるボタンを押しただけなのに送信されてしまった」という報告は、ほぼ必ずこれが原因です。

## 演習課題：社員アンケート画面を作ろう

`index.html` は作りかけのアンケート画面です。

以下の手順でパーツを完成させてください。

> ここでの学習範囲として、サーバーへの送信は行わないため、画面を作るだけで OK です。

### 手順 1: テキストエリアの作成

「意気込み」の欄に、複数行入力できる `<textarea>` タグを配置してください。

その際、`rows="5"` という属性をつけて、最初から 5 行分の高さにしてください。

`<textarea>` は `<input>` と異なり、**閉じタグ（`</textarea>`）が必要**です。

追加後、5 行分の高さの入力欄が表示されれば正解です。

### 手順 2: ラジオボタンのグループ化

「使用している PC」の欄を見てください。

`name` 属性がないため、「Windows」と「Mac」の両方が選択できてしまいます（ラジオボタンなのに！）。

2 つの `<input type="radio">` に同じ `name` 属性（例: `name="pc_os"`）を追加してグループ化してください。

どちらか一方を選んだとき、もう一方が自動的に外れれば正解です。

### 手順 3: label の紐付け（UX 向上）

「興味のある技術」の欄を見てください。

`for` 属性と `id` 属性がないため、「Java」や「Docker」という文字をクリックしてもチェックが付きません。

`<label>` タグに `for` 属性を、対応する `<input>` タグに `id` 属性を追加して紐付けてください（例: `id="tech_java"` と `for="tech_java"`）。

文字をクリックするだけでチェックが付けば正解です。

### 手順 4: セレクトボックスの作成

「所属部署」の欄に、プルダウンメニューを作ってください。`<select>` タグの中に、以下の `<option>` を入れてください。

- 開発部
- 営業部
- 人事部

プルダウンをクリックして 3 つの選択肢が表示されれば正解です。

### 手順 5: バリデーションを体験する

「お名前」の `<input>` に `required` 属性を追加してください。

```html
<input type="text" id="fullname" placeholder="例：山田 太郎" required>
```

追加したら、**お名前を空白のまま** 「回答を完了する」ボタンを押してみてください。

ブラウザが「このフィールドを入力してください」というエラーメッセージを自動で表示すれば正解です。

> **ポイント**: `required` を書くだけで JavaScript は一切不要です。ブラウザが自動でチェックしてくれます。

## 覚えなくて良い・後回しにする内容

以下はフォームに関連する重要なトピックですが、**この段階では覚えなくて構いません**。JavaScript や Angular を学ぶ段階で自然に身につきます。

- **`action` 属性・`method` 属性**: フォームのデータをどこへ、どうやって送るかを指定する属性です。現場では JavaScript/フレームワークが担当するため、素の HTML で書く機会はほぼありません。
- **`<input type="hidden">`**: 画面に表示されない隠し項目。CSRF トークンなどセキュリティ用途で使われますが、フレームワークが自動で処理します。
- **`<fieldset>` と `<legend>`**: フォームをグループ分けするためのタグ。アクセシビリティ向上に役立ちますが、まずは `label` の紐付けを確実に覚えましょう。
- **カスタムバリデーション（`pattern` 属性・JavaScript の Constraint Validation API）**: `required` や `minlength` で足りない複雑なチェックを実装するための仕組みです。

## サーバーへの送信（発展）

このリポジトリでは実際のサーバー送信は扱いません。  
フォーム送信の仕組み（`action` 属性・`method` 属性・HTTP リクエスト）は、JavaScript/Angular を学ぶ段階で一緒に学習します。

> **実務メモ**: Angular では `ReactiveFormsModule` を使ってフォームを管理します。  
> その際も、HTML の `<input>` や `name` 属性の概念がベースになります。

# 04. HTML フォームの部品（UI カタログ）

ここでは、ユーザーからデータを受け取るための様々な「入力部品（パーツ）」の使い方を学習します。
Web アプリの使いやすさ（UX）は、適切なパーツを選べているかで決まります。

## 1. label（ラベル）タグの重要性

フォームを作る時、ただ入力欄 (`<input>`) を置くだけでは不十分です。
「この入力欄は何のためのものか」を示す文字と、入力欄を **紐付ける** 必要があります。

### 紐付けるメリット

文字（ラベル）をクリックしただけで、入力欄にカーソルが当たったり、チェックボックスが `ON` になったりします。これで操作性が劇的に向上します。

> **「操作性が向上する」とは？** <br>
> チェックボックスなどの小さな四角形を正確にクリックするのは、マウスでもスマホのタップでも意外と大変です。 ラベルと紐付けることで **「クリックできる範囲（ヒットエリア）」が文字の分まで広がる** ため、ユーザーがストレスなく操作できるようになります。

```html
<label for="user-name">お名前：</label> <input type="text" id="user-name" />
```

## 2. よく使うパーツ一覧

テキスト入力系
| タグ・属性 | 説明 |
| -- | -- |
| `<input type="text">`| 1 行テキスト <br>普通の文字入力に使います。 |
| `<input type="password">` | パスワード<br>入力した文字が ●●● で隠されます。 |
| `<textarea>` | 複数行テキスト <br>長文の感想やお問い合わせ内容に使います。 |

選択系
| タグ・属性 | 説明 |
| -- | -- |
| `<input type="radio">` | ラジオボタン<br>「A か B か」どれか 1 つを選ばせるときに使います。 同じ name 属性をつけることでグループ化できます。 |
| `<input type="checkbox">` | チェックボックス<br>複数選択の場合に使います。|
| `<select>` & `<option>` | セレクトボックス<br>プルダウン形式で選択肢を表示します。 |

## 演習課題：社員アンケート画面を作ろう

`index.html`は作りかけのアンケート画面です。

以下の手順でパーツを完成させてください。なお、ここでの学習範囲として、サーバーへの送信は行わないため、画面を作るだけで OK です。

### 手順 1: テキストエリアの作成

「意気込み」の欄に、複数行入力できる `<textarea>` タグを配置してください。 その際、`rows="5"` という属性をつけて、最初から 5 行分の高さにしてください。

### 手順 2: ラジオボタンのグループ化

「使用している PC」の欄を見てください。 現在は「Windows」と「Mac」の両方が選択できてしまいます（ラジオボタンなのに！）。 2 つの `<input type="radio">` に 同じ `name` 属性（例: `name="pc_os"`）をつけて、どちらか 1 つしか選べないように修正してください。

### 手順 3: label の紐付け（UX 向上）

「興味のある技術」の欄を見てください。 現在は、四角いチェックボックスそのものをクリックしないとチェックが付きません。 「`Java`」や「`Docker`」という 文字をクリックしてもチェックが付くように、`<label>` タグの for 属性と、`<input>` タグの id 属性を正しく設定してください。

### 手順 4: セレクトボックスの作成

「所属部署」の欄に、プルダウンメニューを作ってください。 `<select> `タグの中に、以下の `<option>` を入れてください。

- 開発部
- 営業部
- 人事部

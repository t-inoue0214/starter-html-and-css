# 08. デザインの基礎（タイポグラフィ・色・余白）

デザイン専門チームが作成した Figma のデザインを、CSS に正確に落とし込むための基礎知識を学びます。「なぜこのフォントサイズ・色・余白なのか」を理解することで、デザイナーとの対話がスムーズになります。

## 1. タイポグラフィ（文字のデザイン）

### rem とは？

CSS でフォントサイズを指定する際、`px` の他に `rem` という単位をよく使います。詳しくは [MDN: CSS の値と単位](https://developer.mozilla.org/ja/docs/Learn_web_development/Core/Styling_basics/Values_and_units) を参照してください。

- `px`：絶対的な大きさ（1px = 画面の1ドット）
- `rem`：ブラウザのデフォルトフォントサイズ（通常 16px）を `1rem` として計算する相対的な大きさ

| rem | px 換算（デフォルト 16px の場合） | 使いどころ |
| :--- | :--- | :--- |
| `0.75rem` | 12px | 小さなラベル・注釈 |
| `0.875rem` | 14px | 補助テキスト・キャプション |
| `1rem` | 16px | 本文（基準） |
| `1.25rem` | 20px | 小見出し |
| `1.5rem` | 24px | 見出し |
| `2rem` | 32px | 大見出し |

> **実務メモ**: Figma の Inspect パネルにはフォントサイズが `px` で表示されます。`14px` と書いてあれば `0.875rem`、`24px` であれば `1.5rem` と変換して実装するのが一般的です（計算式: Figma の px 値 ÷ 16 = rem）。

### よく使うタイポグラフィプロパティ

| プロパティ | 説明 | 実務での値の例 |
| :--- | :--- | :--- |
| [`font-size`](https://developer.mozilla.org/ja/docs/Web/CSS/font-size) | 文字の大きさ | `1rem`（本文）、`1.5rem`（見出し） |
| [`font-weight`](https://developer.mozilla.org/ja/docs/Web/CSS/font-weight) | 文字の太さ | `400`（通常）、`700`（太字） |
| [`line-height`](https://developer.mozilla.org/ja/docs/Web/CSS/line-height) | 行間 | `1.7`（本文）、`1.4`（見出し） |
| [`letter-spacing`](https://developer.mozilla.org/ja/docs/Web/CSS/letter-spacing) | 文字間隔 | `0.05em`（日本語見出しに読みやすさを加える） |

> **なぜ `line-height` を指定するのか**: ブラウザのデフォルト行間（`normal` ≒ 1.2）は、日本語の長い本文には窮屈すぎます。本文は `1.5〜1.7` にするだけで読みやすさが大幅に改善します。

## 2. 色と CSS 変数（デザイントークン）

### CSS カスタムプロパティ（変数）とは？

`--` で始まる名前を使うと、CSS の中で変数（カスタムプロパティ）を定義できます。詳しくは [MDN: CSS カスタムプロパティ](https://developer.mozilla.org/ja/docs/Web/CSS/Using_CSS_custom_properties) を参照してください。

```css
/* :root（HTML 全体）に変数を定義する */
:root {
    --color-primary: #3498db;
    --spacing-md: 16px;
}

/* var() で呼び出して使う */
.btn {
    background-color: var(--color-primary);
    padding: var(--spacing-md);
}
```

> **実務メモ（デザイントークン）**: 実際のプロジェクトでは、ブランドカラーや余白の基準値を CSS 変数として一元管理します。「ブランドカラーを青から緑に変更する」場合、`--color-primary: green;` と 1 箇所を変えるだけで、それを参照しているすべてのボタン・ラベル・リンクの色が一度に変わります。

色の値の書き方については [MDN: color](https://developer.mozilla.org/ja/docs/Web/CSS/color) を参照してください。デザイナーから渡される Figma の仕様には通常 `#3498db` のような16進数で記載されています。

## 3. 余白と視覚的階層

### 余白スケールとは？

「なんとなく余白をつける」と、ページ全体で余白がバラバラになります。現場では `4・8・16・24・40px` のような**倍数ベースのスケール**を使い、一貫したリズムを保ちます。

```css
:root {
    --spacing-xs:  4px;
    --spacing-sm:  8px;
    --spacing-md: 16px;
    --spacing-lg: 24px;
    --spacing-xl: 40px;
}
```

> **実務メモ**: Figma の余白指定は `px` で書かれています。「なぜ 15px ではなく 16px なのか」という疑問はスケールの考え方で解決します。スケールにない値（例: `13px`）が指定されていた場合は、デザイナーに確認するのが正しい対処です。

### 視覚的階層とは？

見る人が「どこが重要か」を直感的に判断できるよう、大きさ・太さ・色・余白を組み合わせます。

| 要素 | `font-size` | `font-weight` | 役割 |
| :--- | :--- | :--- | :--- |
| カテゴリラベル | `0.75rem` | `700` | 分類・タグ（小さくても太く目立つ） |
| 記事タイトル | `1.5rem` | `700` | 最も重要な情報 |
| 日付・読了時間 | `0.875rem` | `400` | 補助情報（小さく控えめに） |
| 本文 | `1rem` | `400` | メインコンテンツ（読みやすさ最優先） |

## 演習課題：デザイン仕様を CSS に落とし込もう

デザイナーから以下のデザイン仕様が届きました。`style.css` を編集して、指定通りの見た目に実装してください。

### デザイン仕様

**カラー**

| 変数名 | 値 | 用途 |
| :--- | :--- | :--- |
| `--color-primary` | `#3498db` | ブランドカラー（ラベル・ボタン） |
| `--color-text` | `#2c3e50` | メインテキスト |
| `--color-text-muted` | `#7f8c8d` | 補助テキスト（日付など） |
| `--color-bg` | `#f0f2f5` | ページ背景 |
| `--color-bg-card` | `#ffffff` | カード背景 |

**余白スケール**

| 変数名 | 値 |
| :--- | :--- |
| `--spacing-xs` | `4px` |
| `--spacing-sm` | `8px` |
| `--spacing-md` | `16px` |
| `--spacing-lg` | `24px` |
| `--spacing-xl` | `40px` |

**テキストスタイル**

| 要素（クラス名） | `font-size` | `font-weight` | `line-height` | その他 |
| :--- | :--- | :--- | :--- | :--- |
| カテゴリラベル（`.post-category`） | `0.75rem` | `700` | — | `letter-spacing: 0.1em` |
| 記事タイトル（`.post-title`） | `1.5rem` | `700` | `1.4` | — |
| メタ情報（`.post-meta`） | `0.875rem` | `400` | — | — |
| 本文（`.post-body`） | `1rem` | `400` | `1.7` | — |
| ボタン（`.btn-read-more`） | `0.875rem` | `700` | — | — |

---

### 手順 1: CSS 変数（デザイントークン）を定義する

`style.css` の `:root` セクションに、上記の**カラー**と**余白スケール**の変数をすべて追加してください。

```css
:root {
    --color-primary: #3498db;
    --color-text: #2c3e50;
    /* ... 残りも追加する */
}
```

DevTools（F12）を開き、`<html>` 要素を選択して Styles パネルを確認します。`--color-primary: #3498db` などの変数が表示されれば正解です。

### 手順 2: タイポグラフィを設定する

デザイン仕様の**テキストスタイル**テーブルを見ながら、各クラスに `font-size`・`font-weight`・`line-height`・`letter-spacing` を設定してください。

この時点では色はまだ当たりません。カテゴリラベルが小さく・タイトルが大きく・本文の行間が広く表示されれば正解です。

### 手順 3: 余白を CSS 変数で設定する

`style.css` 内のコメント「手順 3」の箇所に、`var(--spacing-○○)` を使って `padding` と `margin` を設定してください。

設定する箇所は `.post-card`・`.post-header`・`.post-category`・`.post-title`・`.post-footer`・`.btn-read-more` です。

カード全体に内側の余白が生まれ、要素と要素の間に一定のリズムが見えれば正解です。

### 手順 4: 色を CSS 変数で設定し、変数の威力を確認する

`style.css` 内のコメント「手順 4」の箇所に、`var(--color-○○)` を使って `color` と `background-color` を設定してください。

色が当たったら、試しに `:root` の `--color-primary` の値を `#e74c3c`（赤）に変えてみてください。カテゴリラベルとボタンの色が同時に変われば正解です。確認できたら `#3498db` に戻してください。

## 覚えなくて良い・後回しにする内容

| 内容 | 理由 |
|---|---|
| `em` 単位 | 親要素のフォントサイズを基準にする相対単位。`rem` と混乱しやすいため、今は `rem` と `px` で十分。 |
| `clamp()` による流体タイポグラフィ | 画面幅に応じてフォントサイズが変化する高度な技術。レスポンシブ対応と組み合わせて学ぶ。 |
| HSL / OKLCH 色空間 | 色の明度・彩度で計算する指定方法。デザイナーが色を設計する際の知識。実装では16進数で受け取ることが多い。 |
| Google Fonts の埋め込み | フォント選定はデザイン専門チームの職域。今は `font-family: sans-serif;` のシステムフォントで十分。 |
| `@layer` | カスケードの優先度を層で管理する新しい仕組み。大規模プロジェクト向け。まずは詳細度の基本を習得することが先。 |

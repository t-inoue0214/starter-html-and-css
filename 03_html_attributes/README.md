# 03. HTML の属性 (Attributes)

ここでは、タグに「追加情報」や「設定」を与えるための **属性** について学習します。

## 属性（Attribute）とは？

タグの開始部分 (`< >`) の中に書く `名前="値"` のペアのことです。

そのタグが「どう振る舞うべきか」「どんな性質か」を定義します。

```html
<input type="text" placeholder="ここに入力" />
```

## グローバル属性

どのタグにでも共通して使える属性です。

特に以下の 3 つは Web 開発の最重要項目です。

詳しくは [MDN: グローバル属性](https://developer.mozilla.org/ja/docs/Web/HTML/Global_attributes) を参照してください。

| 属性  | 説明               | ルール                                                                                                                  |
| ----- | ------------------ | ----------------------------------------------------------------------------------------------------------------------- |
| id    | 固有の識別子       | そのページ内で「唯一」の名前をつけます。特定の 1 つを指名したい時に使います。<br> 1 ページに同じ id は 1 回しか使えない |
| class | グループ名         | 複数のタグに同じ名前をつけて、まとめて装飾したり分類したりする時に使います。<br>何回でも同じ class 名を使って OK        |
| style | インラインスタイル | 直接 CSS を書き込みます。動作確認の一時利用には便利ですが、**実務では原則使いません**。<br>理由: 詳細度が最も高く後から CSS で上書きできなくなる・どこにスタイルが書かれているか管理しにくくなるためです。（CSS ファイルに書くのが一般的） |

## CSS ファイルを読み込む：`<link>` タグの属性

`<link>` タグで CSS を読み込む際に使います。

`href` 属性と `rel` 属性は `<a>` タグでも使います。詳しくは後の章で学びます。

| 属性 | 説明                                                                                          | 例                                          |
| ---- | --------------------------------------------------------------------------------------------- | ------------------------------------------- |
| rel  | 関係性 (Relationship) <br> 現在のファイルから見て、読み込むファイルが「何者か」を定義します。 | rel="stylesheet" (これはスタイルシートです) |
| href | 参照先 (Hypertext Reference) <br> ファイルの場所を指定します。                                | href="style.css"                            |

## INPUT タグ固有の属性

入力フォームを作る `<input>` タグは、属性によって姿を大きく変えます。

詳しくは [MDN: input 要素](https://developer.mozilla.org/ja/docs/Web/HTML/Element/input) を参照してください。

| 属性          | 説明                                                                                                       |
| ------------- | ---------------------------------------------------------------------------------------------------------- |
| `type`        | 入力タイプ `text` (文字), `password` (伏せ字), `number` (数値), `radio` (選択) など多数あります。          |
| `name`        | フォーム送信時のキー名。「どのフィールドの値か」をサーバーが識別するための名前です。<br>（例: `name="email"` の入力値は、送信時に `email=xxx` として届きます） |
| `value`       | 初期値。最初から入力欄に入っている値を設定します。                                                         |
| `placeholder` | 入力のヒント。未入力の時にうっすら表示される案内文です。                                                   |
| `disabled`    | 無効化。この属性があると、入力やクリックができなくなります（値も空で指定します）。                         |

## 演習課題：壊れた会員登録フォームを直そう

index.html をブラウザで開くと、真っ白で崩れたフォームが表示されます。

以下の手順で属性を追加し、正しいフォームに修正してください。

### 手順 1: CSS を読み込む（rel 属性）

- `<head>` タグ内にある `<link>` タグを見てください。

    `href` はありますが、`rel` 属性がないため、ブラウザが「これが CSS だ」と認識できていません。

    `rel="stylesheet"` を追加して、画面のデザインが適用されることを確認してください。

`rel="stylesheet"` を追加してページをリロードすると、背景がグレーになりボタンに装飾が付いた状態が表示されれば正解です。

### 手順 2: ID と Class の使い分け

「重要なボタン」と書かれたボタンと、「普通のボタン」があります。

`CSS` ファイルには既に以下の設定が書かれています。

- id が primary-btn のもの → 赤色 になる
- class が normal-btn のもの → 青色 になる

index.html のボタンに正しい属性 (id または class) を追加して、色をつけてください。

赤いボタンが 1 つ・青いボタンが 2 つ表示されれば正解です。

> **実務メモ**: CSS ファイルで `#id` セレクターを使ったスタイリングは、詳細度が高くなりすぎるため実務では避けるチームが多いです。
> この演習は「id と class の違いを体感する」ための例です。実際のコーディングでは class を使ってスタイルを当てるのが一般的です。

### 手順 3: label と input を紐付ける（for / id）

`<label>` の `for` 属性と `<input>` の `id` 属性を同じ値にすることで、ラベルをクリックすると入力欄にフォーカスが当たるようになります。

現在の `index.html` では、`<label>` に `for` 属性がなく、`<input>` に `id` 属性もないため、ラベルをクリックしても何も起こりません。

以下の手順で紐付けを完成させてください。

1. 名前欄の `<label>` に `for="username"` を追加してください。
1. 名前欄の `<input>` に `id="username"` を追加してください。
1. パスワード欄・社員番号欄も同様に `for` と `id` を対応させてください。（値は自由に決めて OK ですが、`for` と `id` が一致していることが必要です）

ラベルをクリックしたとき、入力欄にフォーカスが移動すれば正解です。

### 手順 4: 入力フォームの属性を整える

フォームエリアにある `<input>` タグに必要な属性を追加してください。

1. 名前: ヒント（`placeholder`）として「山田 太郎」と表示させてください。
1. パスワード: 入力した文字が ●●● と伏せ字になるよう、適切な `type` を設定してください。
1. 社員番号: 最初から「`EMP-9999`」という値が入っているように `value` を設定し、かつ書き換えられないように `disabled` にしてください。

社員番号欄に `EMP-9999` が入った状態でグレーアウトされ、編集できなければ正解です。

> **注意**: `disabled` にした入力欄の値はフォーム送信時にサーバーへ届きません。値も送りたい場合は `readonly` を使います。

## コラム：属性の豆知識（発展）

### placeholder の動向

`placeholder` でヒントを表示するのが適切でないという話があります。

- 入力中に情報が消えてユーザーが迷子になる（または、イライラさせる）
- エラー発生時、原因に気づきにくくなる
- 入力済みか未入力か分かりにくい（特に弱視の人）

といったユーザビリティの低下を招くためです。

とはいえ、MDN では非推奨になっているわけではないため、プロダクトの仕様が `placeholder` を必要としている場合には、適切なヒントを表示する必要があります。

### disabled と readonly の違い

| 属性 | 編集 | フォーム送信 |
|:---|:---|:---|
| `disabled` | 不可 | 値が送られない |
| `readonly` | 不可 | 値が送られる |

「値を表示しておきたいが編集させたくない」ケースでは `readonly` が適切なことが多いです。

### 値を書かない（ブーリアン属性）

属性の説明では、`名前="値"` と説明しましたが、実際には `値` を必要としないケースもあります。

`disabled` や `checked` 、 `required` などの属性は、**「書くだけ」** で効果を発揮します。これをブーリアン属性（論理属性）と呼びます。

詳しくは [MDN: 論理属性](https://developer.mozilla.org/ja/docs/Glossary/Boolean/HTML) を参照してください。

```html
<input disabled /> <input disabled="disabled" />
```

> **注意！**  
> `<input disabled="false">` と書いても、「無効化」されます。  
> ブーリアン属性は「値が何か」ではなく **「その属性が存在するかどうか」** だけで判断されるためです。無効化を解除したい場合は、属性自体を削除する必要があります。

### オリジナルの属性を作れる

自分で決めたデータをタグに持たせたい場合は、`data-` で始まる属性名を使います。

詳しくは [MDN: data-*](https://developer.mozilla.org/ja/docs/Web/HTML/Global_attributes/data-*) を参照してください。

```html
<div data-product-id="12345" data-category="book">商品A</div>
```

これは主に `JavaScript` で画面を操作する際に、「このタグには `ID:12345` のデータが紐付いている」と判断させるためによく使われます。

> **なぜ `data-` が必要なの？**  
> 実は `data-` を付けずに記載することも可能ではあります。  
> しかし、`data-` を付けずに好きな名前（例: `memo="..."`）を使うと、もし将来 `HTML` のバージョンアップで公式に `memo` という属性が作られた際、プログラムが誤動作する原因になります。  
> `data-` は「開発者専用の場所」として予約されているため、将来にわたって安全に使えます。

## 学習のヒント：全部覚える必要はありません

HTML の属性は数が膨大ですが、実務で使うものは限られています。

### 1. 今は覚えなくて良いもの（後回し）

- **`itemid`, `itemscope` 等の Microdata 属性**
  - SEO（検索エンジン対策）で使われますが、現在は別の書き方（JSON-LD）が主流になりつつあります。
- **`accesskey`, `contenteditable`, `draggable`**
  - 特定の Web アプリを作る時にしか使いません。

### 2. 使ってはいけないもの（古い書き方）

ネットで検索すると古い情報が出てくることがありますが、以下は **CSS で書くのがルール** です。

- `align="center"` （中央寄せ） -> CSS の `text-align: center;` を使う
- `bgcolor="red"` （背景色） -> CSS の `background-color: red;` を使う
- `<font>` タグすべて

## 覚えなくて良い・後回しにする内容

今の段階で覚える必要はありません。「こういうものがある」程度に知っておくだけで十分です。

| 属性 | 概要 |
|---|---|
| `data-*` | 自分でデータを持たせるカスタム属性。JavaScript と組み合わせて使う。（本章コラム参照） |
| `tabindex` | Tab キーによるフォーカス順序を制御する。アクセシビリティ対応で使うことがある。 |
| `aria-*` | スクリーンリーダー向けのアクセシビリティ属性。実務では重要だが、まずは HTML/CSS の基礎を固めてから。 |
| `contenteditable` | タグの内容をブラウザ上で直接編集可能にする。特殊な UI でのみ使用。 |
